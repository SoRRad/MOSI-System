How It Works — No Token Needed by Surgeons

The platform uses a GitHub Actions workflow as the write layer. Here is the exact flow:

Surgeon submits case
  ↓
Platform calls GitHub API: POST /repos/{owner}/{repo}/actions/workflows/submit_case.yml/dispatches
    (This endpoint only requires a token held by the administrator — baked into the platform at deploy time)
  ↓
GitHub Actions runner (GitHub's servers) picks up the job
  ↓
Runner reads data/cases.json → appends new case → commits → pushes
  ↓
Commit appears in your repository within ~30 seconds

The token in Settings is the administrator's token, stored only in the admin's browser. It triggers the workflow. The workflow itself has permission to write to the repo via GitHub's internal GITHUB_TOKEN — which requires no configuration from surgeons at all.

Result: Surgeons open the URL and submit cases. Nothing else required.
