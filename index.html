How This Works

Every time a surgeon submits a case, the platform:

  1. Saves the case to the browser's localStorage (instant, always works offline)
  2. Fetches the current data/cases.json file from your GitHub repository
  3. Appends the new case to the JSON array
  4. Commits the updated file back to GitHub with a timestamped message
  5. Updates the sync status indicator in the top bar

The Unit Aggregate view always reads from GitHub, so all surgeons see the same pooled data in real time, regardless of which device they used.

If GitHub sync fails (no internet, token expired), the case is still saved locally and flagged for retry. No data is ever lost.
