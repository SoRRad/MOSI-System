How It Works
Architecture, setup guide, and repository structure for the MOSI platform.
⚠ Jekyll Build Error? Create a file named .nojekyll (empty file, exact name with the leading dot) in the root of your repository. This prevents GitHub Pages from running Jekyll, which causes build failures on plain HTML sites.
In GitHub: your repo → Add file → Create new file → name it exactly .nojekyll → leave blank → commit.
ℹ Workflow File Location: The submit_case.yml file must be at .github/workflows/submit_case.yml (with the dot at the start) for GitHub Actions to trigger automatically.

📂 Repository Structure
mosi-system/
├── index.html ← Surgeon platform
├── admin.html ← This admin panel
├── .nojekyll ← Prevents Jekyll build errors (empty file!)
└── .github/workflows/
    └── submit_case.yml ← GitHub Actions (must be in .github/workflows/)
    └── deploy_pages.yml ← GitHub Pages deployment
└── data/
    └── cases.json ← All case records (auto-created by workflow)
🔑 GitHub PAT Permissions Required
When creating a Personal Access Token at github.com/settings/tokens/new, enable the following scopes:
  • repo — full repository access (read/write contents, dispatch workflows)
  • workflow — required to dispatch GitHub Actions workflow_dispatch events
The PAT is stored only in browser localStorage — it is never sent anywhere except directly to api.github.com.
