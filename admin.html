📊 R Analysis Snippets

Load and analyse data directly from GitHub in R:

# Load from GitHub (public repo — no auth needed)
library(jsonlite); library(tidyverse)

url <- "https://raw.githubusercontent.com/owner/mosi-system/main/data/cases.json"
cases <- fromJSON(url) |> filter(!isTest)

# Agreement rates by MOSI stage
cases |> group_by(stage) |>
  summarise(n=n(), stage_agree=mean(stageAgreement=="Agree")*100,
          proc_agree=mean(procAgreement=="Agree")*100)

# Prediction vs ground truth (cases with outcomes)
cases |> filter(!is.na(actualTWL12)) |>
  mutate(predicted_success = recommendedMet,
       actual_success = actualTWL12 >= 20) |>
  group_by(stage) |>
  summarise(calibration = cor(predicted_success, actual_success))
How It Works
Architecture overview for the MOSI Clinical Audit Platform.
🔄 Submission Flow
1
Surgeon opens index.html
No login, no token, no configuration. The admin's GitHub credentials are inherited automatically through shared localStorage.
2
Case submitted
Saved instantly to the surgeon's browser localStorage. This always succeeds, even offline.
3
GitHub Actions workflow dispatched
The platform calls POST /repos/{owner}/{repo}/actions/workflows/submit_case.yml/dispatches using the admin token. This triggers the workflow — surgeons never see the token.
4
GitHub's servers run the workflow (~30s)
The runner reads data/cases.json, appends or updates the case, and commits back using GitHub's internal GITHUB_TOKEN. No external credentials needed.
5
Commit appears in repository
Every submission is permanently versioned. Unit Aggregate reads data/cases.json directly — all surgeons see live pooled data immediately.
📂 Repository Structure
mosi-system/
├── index.html ← Surgeon platform (public-facing)
├── admin.html ← This admin panel (you only)
├── README.md
├── .github/
│   └── workflows/
│       └── submit_case.yml ← GitHub Actions (write layer)
└── data/
    └── cases.json ← All case records (auto-created)
⚠ Workflow File Missing?

If GitHub Actions submissions aren't working, the submit_case.yml workflow file is likely missing. To create it:

  1. In your GitHub repo, click Add file → Create new file
  2. In the filename box, type exactly: .github/workflows/submit_case.yml
  3. Paste the contents of the submit_case.yml file that was generated for you
  4. Click Commit changes
